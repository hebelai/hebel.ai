# Kryptografie & Web of Trust

## Key-Management
- Pro Nutzer ein Keypair (Ed25519/Curve25519).
- Private Keys verbleiben client-seitig.
- Server verwaltet nur Public Keys und Zugriffsrechte.

## Datenfreigabe
- Pseudonyme Ausschreibungen im Pool.
- Selektive Freigaben per QR/Deep-Link-Token (zeitlich begrenzbar).
- Beidseitige Offenlegung erst nach Zuschlag.

## Signierte Nachweise
- Projektabschluss, Bewertungen, Quest-Badges als signierte Assertions (z. B. JSON-LD VC).
